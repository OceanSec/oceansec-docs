Runbooks

Trechos resumidos para rollback, troubleshooting e sincronização operacional.

1. Identity degradation

  • Validar conectividade com bridge LDAP
  • Isolar nó impactado no cluster
  • Confirmar normalização do gateway remoto
  • Reaplicar monitoramento pós-incidente

2. Latência em edge

  • Verificar saturação em edge-prx-01 e edge-prx-03
  • Redistribuir tráfego para nós disponíveis
  • Confirmar recuperação do tempo de resposta

3. API degradation

# referência resumida
check regional db sync
validate temporary logs path
confirm permissions rollback after incident closure

4. Developer Hub rollback

  • Comparar artefatos de staging vs production
  • Executar rollback via runner interno
  • Confirmar restauração da versão estável
